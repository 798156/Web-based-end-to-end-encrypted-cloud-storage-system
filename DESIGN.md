# 基于Web的端对端加密云存储系统设计方案

## 1. 系统概述
本系统旨在实现一个安全可靠的云存储平台，核心特点是**端对端加密（End-to-End Encryption, E2EE）**。这意味着文件在离开用户浏览器之前就已经被加密，服务器端只存储密文，无法窥探用户数据。

## 2. 核心加密流程（关键）
这是毕设中最核心的亮点，必须清晰描述：
1.  **用户注册/登录**：
    *   用户在本地生成一对非对称密钥（公钥/私钥）。
    *   **公钥**发送给服务器公开存储。
    *   **私钥**用用户密码派生的密钥（通过 PBKDF2 或 Argon2 算法）进行对称加密，然后上传服务器保存（方便多端同步，但服务器无法解开私钥）。
2.  **文件上传**：
    *   浏览器为每个文件生成一个随机的**文件密钥**（AES-256）。
    *   使用**文件密钥**对文件内容进行加密（AES-GCM模式）。
    *   使用用户的**公钥**加密**文件密钥**。
    *   将**加密后的文件内容**和**加密后的文件密钥**上传至服务器。
3.  **文件下载**：
    *   下载密文和加密的文件密钥。
    *   用户输入密码解密出**私钥**。
    *   用**私钥**解密出**文件密钥**。
    *   用**文件密钥**解密文件内容。

## 3. 功能模块划分

### 3.1 前端模块 (Web Client)
*   **认证模块**：
    *   注册/登录/注销。
    *   本地密钥生成（Web Crypto API）。
    *   私钥的安全导出与导入。
*   **文件管理模块**：
    *   文件列表展示（解密元数据）。
    *   文件上传（支持拖拽、分片、加密）。
    *   文件下载（流式解密、合并）。
    *   文件删除/移动/重命名。
*   **分享模块**：
    *   生成分享链接。
    *   密钥交换（使用接收者的公钥加密文件密钥）。

### 3.2 后端模块 (Server API)
*   **用户服务**：
    *   用户基本信息管理。
    *   公钥存储库。
    *   加密私钥托管（Vault）。
*   **元数据服务**：
    *   文件目录树结构（虚拟文件系统）。
    *   文件属性（大小、类型、创建时间、加密后的文件名）。
*   **存储服务**：
    *   文件块（Chunk）的存储接口。
    *   对接本地磁盘或对象存储（MinIO/AWS S3/阿里云OSS）。

### 3.3 数据库设计 (Database)
*   **Users 表**：`id`, `username`, `password_hash` (用于验证登录), `public_key`, `encrypted_private_key`, `salt`。
*   **Files 表**：`id`, `user_id`, `parent_id`, `encrypted_name`, `encrypted_file_key`, `file_size`, `storage_path`, `iv` (初始化向量)。
*   **Shares 表**：`id`, `file_id`, `recipient_id`, `encrypted_share_key` (用接收者公钥加密的文件密钥)。

## 4. 技术栈推荐
为了顺利完成毕设，建议选择主流且文档丰富的技术栈。

### 方案 A：Java 全栈（适合学校教学背景）
*   **前端**：Vue 3 + Element Plus + Axios + Web Crypto API。
*   **后端**：Spring Boot + MyBatis-Plus + Spring Security。
*   **数据库**：MySQL + Redis。
*   **存储**：本地磁盘模拟 或 MinIO。

### 方案 B：Node.js 全栈（开发效率高，前后端语言统一）
*   **前端**：React + Ant Design + Web Crypto API。
*   **后端**：NestJS (TypeORM) 或 Koa2。
*   **数据库**：PostgreSQL 或 MongoDB。

### 方案 C：Python 全栈（代码简洁）
*   **前端**：Vue 3。
*   **后端**：Django 或 Flask。
*   **数据库**：MySQL。

## 5. 开发步骤规划
1.  **环境搭建**：配置数据库、后端框架、前端脚手架。
2.  **原型验证**：先写一个简单的 HTML 页面，测试 Web Crypto API 的加密解密功能，确保能在浏览器端跑通 AES 和 RSA 加密。
3.  **后端基础**：实现用户注册登录接口、文件上传下载接口（先不带加密）。
4.  **前后端联调（加密版）**：将前端加密逻辑接入，实现“加密上传”和“解密下载”。
5.  **完善功能**：文件列表、目录结构、文件分享。
6.  **UI美化**：使用组件库优化界面。
7.  **论文撰写**：整理架构图、时序图、核心代码片段。
