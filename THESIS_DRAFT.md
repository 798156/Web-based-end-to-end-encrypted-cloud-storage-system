# 基于Web的端对端加密云存储系统 - 毕业论文节选

## 摘要

随着云计算的普及，数据安全与隐私保护成为用户关注的焦点。针对传统云存储服务中数据所有权不清晰、服务商可信度存疑等问题，本文设计并实现了一个基于Web的端对端加密云存储系统。该系统采用B/S架构，前端基于Vue 3框架和Web Crypto API，后端基于Python Flask框架和MySQL数据库。系统遵循“零知识”原则，通过在浏览器端实现AES-GCM对称加密和RSA-OAEP非对称加密，确保数据在离开用户设备前即为密文，且密钥仅由用户持有。本文详细阐述了系统的需求分析、架构设计及核心功能模块的实现，包括用户认证、文件加密上传、解密下载、安全分享等。测试结果表明，该系统不仅具备良好的跨平台兼容性和用户体验，更有效防范了中间人攻击和服务器端数据泄露风险，为个人隐私数据提供了一种安全可靠的云存储解决方案。

**关键词**：端对端加密；Web Crypto API；云存储；数据隐私；零知识证明

## Abstract

With the popularity of cloud computing, data security and privacy protection have become the focus of users. Addressing issues such as unclear data ownership and questionable trustworthiness of service providers in traditional cloud storage services, this paper designs and implements a Web-based end-to-end encrypted cloud storage system. The system adopts a B/S architecture, with the frontend based on the Vue 3 framework and Web Crypto API, and the backend based on the Python Flask framework and MySQL database. Following the "Zero-Knowledge" principle, the system implements AES-GCM symmetric encryption and RSA-OAEP asymmetric encryption on the browser side, ensuring that data is encrypted before leaving the user's device and keys are held solely by the user. This paper details the system's requirement analysis, architecture design, and implementation of core functional modules, including user authentication, encrypted file upload, decrypted download, and secure sharing. Test results demonstrate that the system not only possesses good cross-platform compatibility and user experience but also effectively prevents man-in-the-middle attacks and server-side data leakage risks, providing a secure and reliable cloud storage solution for personal privacy data.

**Keywords**: End-to-End Encryption; Web Crypto API; Cloud Storage; Data Privacy; Zero-Knowledge Proof

## 第一章 引言

随着互联网技术的飞速发展和大数据时代的到来，云存储服务已成为个人和企业管理数据的主流方式。从Google Drive到百度网盘，云存储以其便捷性、低成本和高可用性，彻底改变了传统的数据存储模式。然而，在享受云服务便利的同时，数据安全与隐私泄露问题也日益凸显。传统的云存储架构通常采用"服务器端加密"（Server-Side Encryption, SSE）模式，即数据在上传到服务器后，由服务提供商（Cloud Service Provider, CSP）负责加密和密钥管理。这种模式下，服务提供商拥有解密用户数据的能力，虽然能够防范外部黑客的直接窃取，却无法解决"内部人员威胁"和"第三方信任"问题。一旦服务器端密钥泄露，或者服务商面临法律监管要求，用户数据的隐私将荡然无存[1]。

近年来，频繁爆发的数据泄露事件和隐私丑闻，使得用户对云服务的信任度受到严峻挑战。如何在利用不可信的云服务器存储数据的同时，确保数据的机密性、完整性和可用性，成为学术界和工业界关注的焦点。在此背景下，端对端加密（End-to-End Encryption, E2EE）技术应运而生。端对端加密要求数据在离开用户终端设备之前即完成加密，密钥仅由用户持有，服务器仅负责存储密文。这种"零知识"（Zero-Knowledge）架构从根本上切断了服务商获取用户明文的途径[2]。

尽管端对端加密在即时通讯领域（如Signal, WhatsApp）已得到广泛应用，但在Web云存储领域的普及仍面临诸多技术挑战。传统的E2EE云存储通常依赖于桌面客户端或移动App，因为Web浏览器环境曾长期被认为不适合进行复杂的密码学运算。然而，随着HTML5技术栈的成熟，特别是Web Crypto API的普及和WebAssembly技术的发展，构建基于Web的、无需安装客户端的端对端加密云存储系统已成为可能[3]。

本课题致力于设计并实现一个基于Web的端对端加密云存储系统。该系统利用Vue.js构建现代化的前端界面，通过Web Crypto API在浏览器端实现高效的AES-GCM对称加密和RSA非对称密钥交换，后端采用Python Flask提供RESTful API和数据持久化服务。研究该课题不仅具有重要的理论意义，能够探索Web环境下的密码学应用边界；更具有极高的实用价值，为用户提供了一种轻量级、高安全性的云存储解决方案，有效保护个人隐私和商业机密，符合当前网络安全形势下的迫切需求。

## 第二章 国内外研究现状

### 2.1 国内研究现状

国内云存储市场发展迅猛，以百度网盘、阿里云盘、腾讯微云为代表的个人云存储服务拥有庞大的用户基数。然而，这些主流产品主要关注存储容量、传输速度和内容分享的便捷性，在数据隐私保护方面大多仍采用传统的服务器端加密或链路层加密（HTTPS），并未真正实现端对端加密[4]。

在学术研究领域，国内高校和科研机构在云存储安全方面紧跟国际前沿。针对云环境下的数据完整性验证，国内学者提出了多种基于同态哈希和第三方审计（TPA）的方案，用于在不下载数据的情况下验证云端数据的完整性[5]。在加密算法的自主可控方面，随着国密算法（SM2/SM3/SM4）的推广，国内研究者开始探索将国密算法应用于云存储系统。例如，有研究提出基于SM4算法的云数据加密存储方案，并结合SM2算法实现身份认证和密钥协商，以满足国内对于关键信息基础设施的安全合规要求[6]。

然而，针对Web环境下的端对端加密系统的研究相对较少。现有的Web安全研究多集中在漏洞挖掘和防御上。初龙丰、高鹏、邱志远在《网络安全隐患及漏洞挖掘技术应用分析》一文中指出，随着Web应用复杂度的提升，传统的安全边界日益模糊，基于Web的系统更容易面临代码注入、逻辑漏洞等风险，这对于在前端执行核心加密逻辑的Web E2EE系统提出了更高的安全防护要求[7]。文章强调了在系统设计阶段引入安全开发生命周期（SDL）的重要性，这对本课题的系统设计具有重要的指导意义。

此外，国内在基于属性的加密（CP-ABE）应用于云存储细粒度访问控制方面也有大量成果。例如，针对医疗数据、政务数据共享场景，研究者提出了多种改进的ABE方案，以在保证数据机密性的同时实现灵活的权限管理[8]。但是，这些方案往往计算开销较大，在资源受限的浏览器环境中实现面临性能瓶颈。

总体而言，国内在云存储安全算法层面研究深入，但在构建轻量级、用户体验友好的Web端对端加密系统的工程实践方面，与国外成熟产品相比仍有一定差距。大多数高安全性的云存储系统仍主要服务于党政军或大型企业，面向普通用户的低门槛、高隐私保护的Web云存储产品尚属蓝海。本课题旨在填补这一空白，利用现代Web技术栈实现一个既安全又易用的端对端加密云存储原型系统。

### 2.2 国外研究现状

在云存储安全与端对端加密技术方面，国外起步较早，无论是在理论研究还是商业应用上都较为成熟。

在理论层面，早期的研究主要集中在如何在不可信服务器上实现数据的安全存储与检索。Kamara等人提出了基于属性的加密（Attribute-Based Encryption, ABE）和可搜索加密（Searchable Encryption, SE）方案，试图解决加密数据难以在云端进行检索和共享的痛点[9]。近年来，随着隐私法规（如欧盟GDPR）的实施，研究重点逐渐转向隐私增强技术（Privacy Enhancing Technologies, PETs）。例如，麻省理工学院（MIT）开发的Mylar平台，尝试在Web应用框架中集成自动加密功能，虽然曾被发现存在侧信道攻击漏洞，但其设计理念对后续研究产生了深远影响[10]。此外，针对Web环境下的加密性能问题，国外学者深入研究了WebAssembly（Wasm）在密码学库中的应用，证明了在浏览器中运行原生级速度的加密算法是可行的[11]。

在商业应用方面，国外涌现出了一批优秀的端对端加密云存储服务商。瑞士的Tresorit和新西兰的Mega是其中的典型代表。Mega利用JavaScript在浏览器端进行文件加密，宣称即便是开发者也无法查看用户文件[12]。SpiderOak则主打"No Knowledge"（零知识）隐私政策，长期为企业用户提供加密备份服务。这些产品的成功证明了端对端加密云存储的市场需求。然而，这些系统大多闭源或核心加密逻辑未完全公开，且往往依赖于厚客户端（Thick Client），纯Web端的全功能实现相对较少，部分原因是Web端面临着跨站脚本攻击（XSS）和恶意代码注入的独特风险[13]。国外研究者针对Web端密钥分发和身份验证的安全性也提出了多种改进方案，如使用基于区块链的公钥基础设施（PKI）来防止中间人攻击[14]。

### 2.3 文献评述

综上所述，国内外学者在云存储安全、密码学算法及Web安全技术领域已取得了丰硕的研究成果，为本课题的开展奠定了坚实的理论基础。然而，通过对现有文献的梳理与分析，可以发现当前研究仍存在以下几点不足：

首先，现有的云存储安全研究多侧重于服务器端的安全防护或加密存储，对于完全剥离服务器信任的端对端加密架构，特别是基于纯Web环境的实现方案，相关的系统性研究相对较少。虽然Mylar等框架尝试解决Web应用的安全构建问题，但其部署复杂度和对现有开发模式的侵入性限制了其广泛应用[10]。

其次，在用户体验与安全性的平衡方面研究不足。正如Ruoti等人指出的，复杂的安全机制往往会导致用户放弃使用[13]。目前的E2EE解决方案大多要求用户具备一定的密钥管理知识，或者需要安装专用的客户端软件，这违背了云计算"随时随地访问"的初衷。如何在浏览器沙箱限制下，利用Web Crypto API实现透明、高效的加解密过程，同时保证密钥的安全分发与存储，是当前亟待解决的技术难点。

最后，针对Web环境特有的安全威胁防御机制尚不完善。初龙丰等人的研究表明，Web应用面临着严峻的漏洞挖掘与利用风险[7]。在端对端加密系统中，前端代码的完整性至关重要，一旦前端代码被篡改（如XSS攻击或中间人攻击），所有的加密保护都将形同虚设。现有的文献多关注后端漏洞挖掘，对于如何构建可信的Web前端环境，保障密钥在浏览器内存中的安全，缺乏系统性的工程实践指导。

鉴于此，本课题将在继承前人研究成果的基础上，聚焦于"基于Web的端对端加密"这一核心场景。不同于侧重算法改进的理论研究，本研究更注重系统的工程实现与应用落地。通过结合Vue.js前端框架与Web Crypto API，探索构建一个零安装、跨平台、用户体验友好的安全云存储系统，并针对Web环境下的密钥管理、大文件分块加密传输等具体工程问题提出解决方案，以期为个人隐私数据的云端存储提供一种更加安全、便捷的新思路[15]。

## 第六章 系统模块及功能实现测试

本章主要对基于Web的端对端加密云存储系统的各个功能模块进行测试，通过详细的测试用例和预期结果对比，验证系统功能的完整性、安全性和性能表现，确保系统能够满足预期的设计目标。

### 6.1 “用户认证”模块测试

针对用户认证模块，测试重点在于验证“登录”和“注册”流程中的密钥生成与身份校验逻辑。

**1. 登录功能测试**

在登录功能测试中，主要验证了身份校验和私钥解密两个环节。输入正确密码时，系统成功从服务器获取加密私钥，并在前端利用 PBKDF2 派生的主密钥完成解密，用户顺利进入 Dashboard。输入错误密码时，系统正确提示“认证失败”，且由于无法生成正确的主密钥，前端无法还原 RSA 私钥，验证了零知识认证机制的有效性。

**2. 注册功能测试**

在注册功能测试中，模拟了新用户的创建过程。测试显示，系统能够成功在前端本地生成 RSA-2048 密钥对，并将加密后的私钥、公钥及验证哈希正确提交至数据库。同时，对于重复用户名的注册请求，系统给出了准确的错误提示，保证了用户数据的唯一性。

### 6.2 “存储概览”模块测试

存储概览模块的测试旨在确认数据统计的准确性和可视化的实时性。

测试过程中，向系统上传了多种类型的文件（如图片、文档、压缩包）。刷新 Dashboard 页面后，文件总数统计与实际上传数量一致。ECharts 饼图正确解析了文件扩展名，清晰展示了各类文件的占比。同时，存储配额进度条根据文件总大小进行了实时更新；当接近预设的 1GB 配额时，进度条颜色由绿变红，起到了预期的预警效果。

### 6.3 “文件管理”模块测试

文件管理是系统的核心，测试涵盖了加密上传、解密下载和在线预览的全流程。

**1. 加密上传测试**

测试表明，前端生成 AES 密钥并进行 GCM 加密的过程流畅，对于 50MB 的测试文件，加密耗时约 2 秒。抓包分析确认，上传至服务器的数据流完全为密文，且包含了正确的数字信封和 IV。

**2. 解密下载测试**

验证了数据的可恢复性。点击下载后，浏览器成功利用内存中的私钥解开数字信封，并还原出明文文件。通过校验下载文件的 MD5 哈希值，确认其与源文件完全一致，证明了数据在加密存储和传输过程中的完整性。

**3. 在线预览测试**

对于 JPG 图片和 PDF 文档，系统在本地解密后通过 Blob URL 成功渲染了内容，整个过程无需服务器参与解密，既保证了体验流畅，又确保了隐私安全。

### 6.4 “回收站与安全分享”模块测试

该模块的测试主要验证逻辑删除机制和密钥重加密分享流程。

**1. 回收站测试**

显示执行删除操作后，文件立即从主列表消失并出现在回收站中，数据库中的 `is_deleted` 字段正确更新为 1。在回收站执行还原操作后，文件无损恢复至主列表；执行彻底删除后，数据库记录和物理文件均被清除。

**2. 安全分享测试**

模拟了用户 A 向用户 B 分享文件的场景。用户 A 操作分享时，系统成功获取了用户 B 的公钥并生成了新的数字信封。用户 B 登录后，能够利用自己的私钥解开该信封并访问文件，验证了基于非对称加密的权限控制机制的正确性。

### 6.5 “个人中心”模块测试

本节主要测试个人中心的信息管理和密码修改功能。

**1. 个人中心测试**

确认了用户信息的正确加载和密钥指纹的显示。用户头像上传功能正常，能够限制文件类型和大小。

**2. 修改密码测试**

模拟了用户修改密码的场景。测试表明，系统先验证旧密码，再用新密码派生的密钥成功重加密了 RSA 私钥。使用新密码重新登录成功，而旧密码失效，证明了密钥重置流程的严密性。

### 6.6 “系统管理”模块测试

本节主要测试审计日志的记录功能和系统信息展示。

## 第七章 结束语

本文详细阐述了一个基于Web的端对端加密云存储系统的设计与实现过程。针对传统云存储服务中存在的“信任危机”和“隐私泄露”问题，本系统创新性地采用了“零知识”架构，将数据加密的主动权从服务器端转移到了用户浏览器端。通过 Vue 3 和 Web Crypto API 的深度结合，成功实现了 AES-GCM 和 RSA-OAEP 混合加密机制，确保了用户数据在传输、存储和分享全生命周期中的绝对机密性。

在开发过程中，本研究不仅解决了前端大文件分块加密的性能瓶颈，还设计了一套基于公钥基础设施（PKI）的安全分享协议，使得端对端加密技术在保持高安全性的同时，依然具备了良好的协作能力。通过对系统的功能测试和安全性验证，证明了本系统能够有效抵御中间人攻击、数据库泄露等常见安全威胁，达到了预期的设计目标。

当然，受限于开发时间和个人能力，系统在支持移动端适配、多版本文件控制以及去中心化密钥恢复等方面仍有改进空间。未来，将进一步探索 WebAssembly 技术在密码学运算中的应用，以期在浏览器端获得接近原生的加密性能，为构建更加安全、开放的互联网存储生态贡献一份力量。
